## Applying Migrations

### To Latest

```bash
# Apply all pending migrations
alembic upgrade head

# Apply to specific version
alembic upgrade 003
```

### Rollback

```bash
# Rollback one migration
alembic downgrade -1

# Rollback to specific version
alembic downgrade 002
```

### Check Status

```bash
# Show current version
alembic current

# Show migration history
alembic history

# Show pending migrations
alembic heads
```

## Creating New Migrations

### Autogenerate from ORM

```bash
# Compare ORM models to database, generate migration script
alembic revision --autogenerate -m "description_of_change"
```

### Manual (for complex operations)

```bash
# Create empty migration
alembic revision -m "description_of_change"
```

### Best Practices

1. **Always review autogenerated migrations** — they may miss or misinterpret some changes
2. **Use `CONCURRENTLY` for index creation** — avoids table locks
3. **Batch large data migrations** — use `ALTER TABLE ... ALTER COLUMN ... TYPE ... USING` instead of drop-recreate
4. **Test on staging first** — run `alembic upgrade head` on a staging database
5. **Write `downgrade()`** — always ensure reversibility (except destructive operations)

## Migration Process

### Development

```bash
# 1. Modify ORM models in src/database/models/
# 2. Generate migration
alembic revision --autogenerate -m "my_change"
# 3. Review and edit the generated file
# 4. Apply
alembic upgrade head
# 5. Verify
alembic check  # Ensures ORM matches DB
```

### Staging

```bash
# 1. Deploy migration files
# 2. Run dry-run first
alembic upgrade head --sql > dry_run.sql
# Review dry_run.sql for correctness

# 3. Apply
alembic upgrade head
```

### Production

```bash
# 1. Take database backup
pg_dump -Fc football_prediction > backup_$(date +%Y%m%d).dump

# 2. Apply with care
alembic upgrade head

# 3. Verify
alembic check

# 4. Rollback if issues
alembic downgrade -1
```

## Alembic Configuration

```python
# alembic/env.py

from src.database.models import *  # noqa — imports all models for autogenerate
from src.database.base import Base

target_metadata = Base.metadata

def run_migrations_online():
    from src.database.session import create_engine_from_config
    engine = create_engine_from_config()
    with engine.connect() as connection:
        context.configure(
            connection=connection,
            target_metadata=target_metadata,
            compare_type=True,       # Detect type changes
            compare_server_default=True,  # Detect default changes
        )
        with context.begin_transaction():
            context.run_migrations()
```

## Common Migration Patterns

### Adding an Index (Lock-Safe)

```python
def upgrade():
    op.create_index(
        "ix_matches_new_column",
        "matches",
        ["new_column"],
        postgresql_concurrently=True,
    )
```

### Changing Column Type

```python
def upgrade():
    op.alter_column(
        "odds",
        "odds_home",
        type_=sa.Numeric(6, 2),
        postgresql_using="odds_home::numeric(6,2)",
    )
```

### Adding a Partial Index

```python
def upgrade():
    op.create_index(
        "ix_odds_pinnacle",
        "odds",
        ["match_id"],
        postgresql_where=sa.text("source = 'Pinnacle'"),
        postgresql_include={"odds_home", "odds_draw", "odds_away"},
    )
```

## Data Migration Patterns

### Batch Migration (for large tables)

```python
def upgrade():
    # 1. Create new partitioned table
    op.execute("""
        CREATE TABLE matches_new (
            LIKE matches INCLUDING DEFAULTS INCLUDING CONSTRAINTS
        ) PARTITION BY RANGE (match_date)
    """)
    
    # 2. Create partitions
    op.execute("""
        CREATE TABLE matches_2024 PARTITION OF matches_new
        FOR VALUES FROM ('2024-01-01') TO ('2025-01-01')
    """)
    
    # 3. Migrate data in batches
    connection = op.get_bind()
    connection.execute("""
        INSERT INTO matches_new SELECT * FROM ONLY matches
        WHERE match_date >= '2024-01-01' AND match_date < '2025-01-01'
    """)
    
    # 4. Swap tables
    op.execute("ALTER TABLE matches RENAME TO matches_old")
    op.execute("ALTER TABLE matches_new RENAME TO matches")
    op.execute("DROP TABLE matches_old")
```
